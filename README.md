Este proyecto revolucionó el control de calidad en entornos de educación superior al conectar directamente el área de proyectos con el núcleo académico, sustituyendo revisiones manuales extenuantes por un sistema integral de auditoría algorítmica y multimodal. La solución descarga de manera segura grabaciones desde Google Drive, las procesa mediante compresión y aceleración heurística para optimizar costos, y delega el análisis pedagógico y técnico a la IA de Google Gemini, logrando extraer evaluaciones objetivas sobre el desempeño docente, el nivel de participación y el dominio del tema. El impacto en el negocio fue transformador: el sistema generó la capacidad de optimizar el tiempo de auditoría de los videos en un 96.67%, produciendo reportes estandarizados en formatos JSON, CSV y PDF de alta calidad que democratizan la retroalimentación institucional.

A nivel arquitectónico, el sistema es un ejemplo de resiliencia y manejo avanzado de recursos. Implementa un robusto mecanismo de contingencia (*fallback*) iterativo entre MoviePy y FFmpeg nativo a través de subprocesos, garantizando el procesamiento de archivos pesados y mitigando cuellos de botella en la memoria del servidor. Además, para sortear los estrictos límites de tokens de contexto de la Inteligencia Artificial, el código orquesta una segmentación dinámica del contenido de video, integrando un mecanismo de memoria acumulada (inspirado en la arquitectura RAG) que inyecta los descubrimientos de fragmentos anteriores como contexto para los siguientes, garantizando así un informe estructuralmente coherente de principio a fin.

```python
# Segmentación y compresión de video robusta por subprocesos
subprocess.run(['ffmpeg', '-i', video_path, '-filter_complex', '[0:v]setpts=0.666667*PTS...', '-segment_time', '1500', output])

# Lógica de memoria acumulada para auditoría de segmentos con Gemini
contexto_acumulado = ""
```

La gestión de la deuda técnica y la documentación en metodologías ágiles representa un desafío persistente que suele devorar recursos críticos del equipo de ingeniería. Este orquestador autónomo erradica ese cuello de botella interactuando directamente con la API de GitHub para iterar sobre repositorios, extraer recursivamente la lógica de negocio de los archivos fuente (`.py`, `.sql`, `.ipynb`, `.json`) y generar una redacción exhaustiva de manuales técnicos, arquitecturas y dependencias utilizando el poder analítico combinado de modelos como Llama-3, DeepSeek y Gemini. Lo que antes demandaba que proyectos complejos se documentaran a lo largo de múltiples días de laboriosa redacción manual, hoy permite contextualizar y documentar de manera bilingüe (español e inglés) la totalidad del flujo de datos en escasos 10 minutos por iteración.

Técnicamente, el script funciona como un agente autónomo que controla su propio estado a través de un *log* histórico (archivo TXT) para identificar repositorios previamente procesados, previniendo reprocesamientos redundantes y optimizando quirúrgicamente las cuotas de peticiones a las APIs. Se implementaron intrincados motores de *fallback* y control de excepciones mediante esperas de tiempo (`time.sleep`) que evaden las interrupciones por saturación de red, asegurando la entrega atómica de la documentación final mediante *commits* programáticos directos en las ramas de producción de GitHub.

```python
# Búsqueda recursiva para extraer código fuente de archivos clave
acumulated_script = ""
elementos = repo.get_contents("")
while elementos:

    if archivo.type == "dir":
        elementos.extend(repo.get_contents(archivo.path))
    elif archivo.name.endswith(('.py', '.sql', '.ipynb', '.json')):
        acumulated_script += f"\n\n### {archivo.path} ###\n{archivo.decoded_content.decode('utf-8')}"
```

### [4. Enterprise Analytics Pipeline & Ingesta Centralizada de Datos (BigQuery)](https://github.com/SGC140/BigQuery-Project---ETL-Data-Analysis-)
![BigQuery](https://img.shields.io/badge/BigQuery-Data_Warehouse-blue?style=flat-square) ![SQL](https://img.shields.io/badge/SQL_ANSI-Transformations-orange?style=flat-square) ![Looker Studio](https://img.shields.io/badge/Looker_Studio-BI-4285F4?style=flat-square) ![Power BI](https://img.shields.io/badge/Power_BI-BI-F2C811?style=flat-square)

Esta ambiciosa arquitectura resolvió la alta fragmentación y el aislamiento de datos operativos, integrando bases provenientes de múltiples proyectos independientes de impacto social y educativo (Ecolombia, Jóvenes a la E, Suba) en un *Data Warehouse* centralizado bajo el entorno de Google Cloud Platform. A nivel de negocio, este esfuerzo unificó las vistas analíticas, nutrió *dashboards* directivos en Power BI y Looker Studio, y optimizó dramáticamente el tiempo dedicado a pensar estrategias en lugar de limpiar datos manualmente. Sus modelos SQL de trazabilidad habilitaron mecanismos de seguimiento temprano, conciliación de telemetría y perfiles de empleabilidad que fueron determinantes para lograr una retención académica del 97% y una tasa de aprobación del 80% en los proyectos focalizados.

Las consultas empresariales alojadas en BigQuery se diseñaron bajo el paradigma ELT (Extraer, Cargar, Transformar), minimizando drásticamente los costos de procesamiento de Google Cloud a través del uso de funciones de ventana analíticas (`QUALIFY ROW_NUMBER()`), desanidamiento avanzado de estructuras dinámicas y anidadas (`UNNEST` con `STRUCT`), y complejas reglas de homogeneización de cadenas textuales con Expresiones Regulares. Este diseño unifica de manera magistral variables socioeconómicas y genera diagnósticos preventivos en materia de progreso académico.

```sql
-- Normalización sociodemográfica y deduplicación analítica a bajo costo de escaneo
SELECT
  DOCUMENTO,
  CASE 
    WHEN REGEXP_CONTAINS(LOWER(ESCOLARIDAD), 'universitario|pregrado') THEN 'Universitario Completo'
    ELSE ESCOLARIDAD 
  END AS NIVEL_EDUCATIVO
FROM `proyecto_gcp.dataset.raw_data`
QUALIFY ROW_NUMBER() OVER(PARTITION BY DOCUMENTO ORDER BY FECHA_ACTUALIZACION DESC) = 1
```
